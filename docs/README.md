# Índice de documentación

Esta carpeta contiene los documentos de trabajo del proyecto. Usa el siguiente índice para saber **qué completar**, **para qué sirve** y **en qué orden**.

## Orden recomendado de llenado
1. `../PROJECT_BRIEF.md`
2. `design_concept.md`
3. `implementation_plan.md`
4. `architecture_decision.md`
5. `decision_log.md`
6. `ci.md`

10. `visual_regression.md`

## Archivos y propósito
| Archivo | Propósito |
| --- | --- |
| `../PROJECT_BRIEF.md` | Brief del proyecto: contexto, objetivos, alcance y restricciones. |
| `design_concept.md` | Tesis de diseño, patrones clave y dirección visual. |
| `implementation_plan.md` | Plan de ejecución: tareas, hitos y dependencias. |
| `architecture_decision.md` | Decisiones de arquitectura y trade-offs clave. |
| `decision_log.md` | Registro de decisiones tomadas durante el proyecto. |
| `ci.md` | Configuración y checklist de CI/CD. |
| `security_checklist.md` | Checklist de seguridad aplicada. |
| `threat_model.md` | Modelo de amenazas y mitigaciones. |
| `ui_audit.md` | Auditoría de UI (consistencia, accesibilidad, estándares). |
| `visual_regression.md` | Plan y resultados de regresión visual. |
| `design_qa.md` | QA de diseño: validaciones finales antes de release. |
| `walkthrough.md` | Recorrido guiado del producto para stakeholders. |
| `release_checklist.md` | Checklist de release (QA, seguridad, comunicación). |
