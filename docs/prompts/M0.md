# M0: Diseño Anti-Genérico

## OBJETIVO (OBJ)
Definir la identidad visual y estratégica para evitar el "Look & Feel de IA". Establecer una tesis de diseño clara y patrones de firma que diferencien el producto final.

## ALCANCE (ALC)
- Documentación de tesis de diseño.
- Definición de 3 "Signature Patterns" (patrones únicos).
- Estrategia de contenidos y assets para evitar genéricos.
- Actualización de `design_library.template.md`.

## PASOS
1.  **Crear `docs/design-thesis.md`**:
    - Definir los "Valores de Diseño": Asimetría controlada, tipografía con carácter, micro-interacciones con propósito.
    - Listar "Banned Patterns" (gradientes por defecto, layouts de 3 columnas centradas, ilustraciones flat de stock).
2.  **Definir 3 Signature Patterns**:
    - Crear `docs/signature-patterns.md`.
    - Patrón 1: Navegación inmersiva o "Mega Menu" con identidad.
    - Patrón 2: Layout de "Bento Grid" dinámico para features/servicios.
    - Patrón 3: Footer narrativo o "Site Map" visual.
    - Incluir especificaciones de a11y (accesibilidad) y `prefers-reduced-motion`.
3.  **Estrategia de Assets Anti-IA**:
    - Añadir sección en `design-thesis.md` sobre "Imperfecciones Humanas".
    - Reglas para fotografía: Usar ruido suave, recortes no estándar, evitar "personas sonriendo a la cámara" de stock gratuito.
    - Reglas para Copy: Prohibir palabras como "innovación", "soluciones integrales" sin contexto específico.

## CRITERIOS DE ACEPTACIÓN (AC)
- [ ] `docs/design-thesis.md` creado y validado.
- [ ] 3 Signature Patterns documentados con ejemplos de estructura HTML/CSS (conceptuales).
- [ ] Guía de "Banned Patterns" incluida.
- [ ] No requiere código funcional todavía, solo definición estratégica.

## EJECUCIÓN (EJ)
```bash
# Crear archivos de documentación
touch docs/design-thesis.md docs/signature-patterns.md
```

## RIESGOS
- Diseñar patrones demasiado complejos que afecten el rendimiento (CLS/LCP).
- Abstraer demasiado la "tesis" y que no sea accionable.
