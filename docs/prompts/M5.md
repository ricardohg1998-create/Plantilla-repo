# M5: SEO Técnico

## OBJETIVO (OBJ)
Asegurar que la web sea perfectamente indexable y compartible, cumpliendo lso requisitos técnicos de Google y redes sociales.

## ALCANCE (ALC)
- Robots.txt, Sitemap.xml.
- Meta Tags, Open Graph, Twitter Cards.
- Canonical URLs, 404 custom.
- Schema.org básico.

## PASOS
1.  **Meta & OG**:
    - Configurar `metadata` API (Next.js) o `react-helmet-async` (Vite) para generar tags dinámicos.
    - Asegurar que TODAS las páginas tienen: Title, Description, OG Image (generada o default), OG Title.
2.  **Archivos Raíz**:
    - Generar `robots.txt` permitiendo todo (salvo admin/api).
    - Generar `sitemap.xml` dinámico para blog/productos.
3.  **Estructura Técnica**:
    - Validar etiquetas canónicas (self-referencing por defecto).
    - Crear página 404 personalizada y útil (con buscador o links a home).
    - (Opcional) Feed RSS para el blog.
4.  **Schema.org**:
    - Implementar JSON-LD para `LocalBusiness` o `Organization` en la home.
    - Implementar `Article` para posts de blog.

## CRITERIOS DE ACEPTACIÓN (AC)
- [ ] Auditoría SEO (Lighthouse/Ahrefs) sin errores graves técnicos.
- [ ] Previsualización de links en Twitter/WhatsApp/LinkedIn muestra la imagen y titulos correctos.
- [ ] `sitemap.xml` se actualiza con nuevas rutas.

## EJECUCIÓN (EJ)
Uso de `next-sitemap` para generación automática post-build en Next.js.
Script custom para Vite.

## RIESGOS
- Contenido duplicado si la gestión de canonicals falla (ej: variantes de URL con params).
- Bloquear googlebot accidentalmente en staging y olvidar quitarlo en prod.
