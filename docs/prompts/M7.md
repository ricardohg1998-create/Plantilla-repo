# M7: Workflows & Gates

## OBJETIVO (OBJ)
Limpiar la carpeta de workflows de "basura automática" y establecer controles reales y útiles que impidan romper producción.

## ALCANCE (ALC)
- Depuración de `.agent/workflows`.
- Refactorización de scripts de mantenimiento.
- Actualización de `README.md` operativo.

## PASOS
1.  **Limpieza de Workflows**:
    - Eliminar o marcar DEPRECATED cualquier workflow que haga "magia negra" sin control.
    - Consolidar en:
        - `/kickoff`: Inicia proyecto (pregunta brief, elige stack, limpia repo).
        - `/scaffold`: Crea nueva página/sección respetando arquitectura.
        - `/qa_polish`: Auditoría visual final.
        - `/release_proof`: El "Gatekeeper" final.
2.  **Implementación de Gates**:
    - El script `/release_proof` debe ejecutar:
        1. Typecheck (tsc).
        2. Linting (eslint).
        3. Tests unitarios (vitest).
        4. Content Check (no pending items).
        5. Build Check (build real).
    - Si algo falla, el release se detiene.
3.  **README Operativo**:
    - El `README.md` raíz debe ser para el DESARROLLADOR (Tú/Agente), explicando cómo usar los comandos.

## CRITERIOS DE ACEPTACIÓN (AC)
- [ ] Workflow `/release_proof` falla si hay errores de TS o contenido draft.
- [ ] Workflows redundantes eliminados.
- [ ] `README.md` explica claramente cómo arrancar un proyecto nuevo desde cero.

## EJECUCIÓN (EJ)
```markdown
# README
## Comandos del Agente
- `/kickoff`: Iniciar nuevo cliente.
- `/release_proof`: Verificar antes de entregar.
```

## RIESGOS
- Crear gates tan lentos que se omitan manualmente. (Mantener checks < 2 min).
