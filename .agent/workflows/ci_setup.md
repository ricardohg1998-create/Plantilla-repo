> ⚠️ Este workflow es OPCIONAL. No es necesario para webs corporativas simples.
> Úsalo si el proyecto tiene backend, múltiples desarrolladores, o el cliente lo requiere.

1. Configurar pipeline de CI
   Añadir workflow de GitHub Actions con: install, lint, typecheck, build.
   Usar la skill `ci-cd-hardener` como guía.

2. Documentar
   Crear `docs/ci.md` con los comandos y su descripción.
