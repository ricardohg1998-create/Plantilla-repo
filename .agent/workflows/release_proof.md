1. Ejecutar checklist de release
   Crear `docs/release_checklist.md` verificando:

   **Calidad visual:**
   - [ ] La web NO parece hecha con un template genérico.
   - [ ] Los patrones de firma están implementados y son visibles.
   - [ ] La tipografía tiene personalidad y no es la default del framework.
   - [ ] Los colores son intencionados y alineados con la marca.
   - [ ] No hay placeholders ni contenido genérico.

   **Funcionalidad:**
   - [ ] Todas las páginas navegan correctamente.
   - [ ] Los formularios funcionan y validan.
   - [ ] Responsive correcto en desktop y móvil.
   - [ ] Las animaciones funcionan sin errores.

   **Técnico:**
   - [ ] Build pasa sin errores.
   - [ ] Imágenes optimizadas.
   - [ ] SEO básico implementado (títulos, metas).

2. Documentar
   Actualizar `docs/walkthrough.md` y `docs/decision_log.md` con evidencia final.
